## 🏋️‍♀️ 직원 (관리자)

### 스케줄 조회

![schselect](https://user-images.githubusercontent.com/99962500/197467821-5c8e1d39-be61-46a4-877e-0bb98d1873fe.gif)

 - 전 직원 예약 스케줄 조회 (SELECT)
    
   - Fullcalendar API 사용

   - 메뉴바에서 <스케줄 조회> 목록 클릭 시 전 직원 스케줄 조회 가능

   - 상단에 로그인한 직원 이름 출력됨

   - 직원 이름, 수업 시간 확인 가능

   - 월별, 주별, 일별로 조회 가능 / 일정목록의 경우 해당하는 한 주의 모든 스케줄을 요일별로 확인 가능 

   - 일자 클릭 시 일별 조회로 넘어감 

   - 하루에 스케줄이 3개 이상일 경우 하단에 +n 형태로 보여짐 

### 회원 관리

#### 회원 조회

![userselect](https://user-images.githubusercontent.com/99962500/197468684-a1e16827-3b84-4238-be3d-28e6532c55b3.gif)

- 회원 전체조회 (SELECT)

   - 메뉴바에서 <회원관리 - 회원 조회> 클릭 시 전 회원 조회 가능

   - 이름, 회원번호, 프로필, 전화번호, 등록일, 만료일, 해당 회원의 이용권 확인 가능

   - 프로필은 회원 등록 시 등록한 사진이 보여지고 등록하지 않은 경우 기본 아바타 사진이 보여짐 

   - 프로필에 커서를 올리면 사진에 동작이 부여되면서 성별(남성 : M / 여성 : F) 확인 가능 

   - PT 등록한 경우 등록 버튼이 활성화되어 등록 페이지로 이동하고 이미 수업을 등록한 회원인 경우 등록완료로 보여지며 버튼 비활성화 됨
 
 - 회원 상세조회 (SELECT)
   
   - 전체 조회표에서 hover 기능을 통해 해당 회원 상세페이지로 이동함 
   
 ![userupdate](https://user-images.githubusercontent.com/99962500/197472367-ca2ca365-4b32-4f30-99c5-42ebe305323a.gif)
 
 - 회원 수정 (UPDATE)

    - 이름, 전화번호, 생년월일, 성별, 회원권, 프로필만 수정 가능함

    - 회원 등록 시 프로필을 등록하지 않은 회원은 기본 아바타 사진이 보여지고 클릭 시 사진 등록 가능 

    - PT 이용권을 등록한 경우 등록일만 보여지고 헬스장 이용권의 경우 등록일, 구매한 이용 개월권, 만료일 확인 가능 

    - PT에서 헬스장 이용권으로 변경할 경우 이용권 테이블로부터 조회하여 DB에 존재하는 이용권만 보여지며 선택할 수 있음
      
![userdelete](https://user-images.githubusercontent.com/99962500/197474114-cee90ab0-9424-458c-855d-cc9ddfe24a53.gif)

 - 회원 삭제 (UPDATE)

    - 회원 삭제 버튼 클릭 시 해당 회원의 status가 'N'으로 update되면서 조회 페이지에 출력되지 않음 
      
![search](https://user-images.githubusercontent.com/99962500/197469990-c838e02c-12e1-4420-aa48-72f28db9e7d9.gif)
 
 - 검색 (이름, 이용권)
 
    - 이름과 이용권 2가지 방법으로 검색 가능
   
    - 이름의 경우 '김'으로 검색하면 '김'이 포함된 회원들만 나오고, '김제니'라고 검색하는 경우 해당 회원만 검색됨 
  
    - 이용권의 경우 'P'와 'H'로 검색하면 해당 이용권을 등록한 회원만 검색됨 
  
#### 회원 등록 

 ![userenroll](https://user-images.githubusercontent.com/99962500/197473733-d6d00967-7df2-4d26-a0af-a7301753b49f.gif)

 - 회원 등록 (INSERT)
 
    - 메뉴바에서 <회원관리 - 회원 등록> 클릭 시 등록 페이지로 이동

    - 신규 회원 등록 시 프로필을 제외하고 모두 필수 값 : 이름, 전화번호, 생년월일, 성별, 회원권, 등록일, 이용개월권 

    - 전화번호의 경우 회원이 로그인할 때 아이디로 사용되기 때문에 중복검사 진행 

    - 전화번호 11자리 미만, 초과한 경우 또는 이미 등록된 번호인 경우 '올바른 전화번호를 입력해주세요' 문구 출력 됨

    - 전화번호는 11자리에 등록되지 않은 번호인 경우만 '회원가입 가능한 전화번호입니다' 문구 출력 됨

    - PT 이용권인 경우 등록횟수를 PT 수업 페이지에서 별도로 선택하기 때문에 등록일만 기입

    - 헬스장 이용권인 경우 등록일과 이용권 테이블로부터 조회하여 DB에 존재하는 개월권만 보여지며 선택 가능 

    - 등록버튼 클릭 시 모달을 통해 다시 한번 확인 후 모달의 확인 버튼 클릭 시 alert로 등록되었다는 문구와 함께 회원 전체조회 페이지로 넘어감

#### 스케줄 조회

![uschselect](https://user-images.githubusercontent.com/99962500/197930390-6cebbc75-cd60-4d34-990e-669d86413e85.gif)

 - 나의 예약 및 담당 트레이너 스케줄 조회 (SELECT)

## 회고

- 좋았던 점

  세미프로젝트 당시에는 시간이 부족하여 기본 기능을 구현하는 것에만 집중하다보니 다양한 기능을 사용하지 못 했던 아쉬움이 많았다. 
  이번에는 기능별로 마감일을 정하여 체계적으로 스케줄 관리한 덕에 기간 내에 프로젝트를 완수 할 수 있었고, 세미프로젝트에서 사용하지 못 했던 Ajax와 open API를 사용함으로써 같은 기능도 다양한 방법으로 구현이 된 다는 것을 몸소 느끼게 되었다. 
  또한 세미프로젝트 당시에는 오류를 접하면 무엇이 문제이고 이를 해결하기 위해 왜 이러한 방식을 선택했는지 보다 구글링을 통해 단순히 해당 오류를 빠르게 해결하려고만 했다면, 파이널을 통해 기능 구현을 할 때 내가 풀어가야 할 문제의 본질을 충분히 파악한 후에 접근하여 단계별로 해결해야 오류를 접하더라도 어느 부분에서 오류가 발생했는지 알기 쉽다는 것을 느꼈다. 또한 동료들에게 물어보며 하나의 문제를 다양한 시각으로 바라볼 수 있는 시간을 갖게 되어 좋았다.
  
- 아쉬운 점

  사용자 측 회원 수업 예약, 변경 부분에서 해당 날짜와 시간을 DB에 존재하는 데이터를 SELECT 한 후에 이를 제외한 부분만 선택할 수 있도록 해야 사용자 입장에서 등록과 변경 기능을 수월하게 사용할 수 있는데 조건처리 부분이 미흡하여 사용에 약간의 불편함이 있다. 또한 이용권 수정 부분에서 수정 모달을 반복문 안에 넣어 구현하여 코드가 지져분해져 아쉬움이 많았다. 단순히 기능을 구현하는 것에 멈추지 않고 어떻게 하면 사용자 입장에서 더 쉽고 간편하게 해당 기능을 사용할 수 있는지 생각해야하고 구현이 1차적으로 완성되어 기능을 한다면 그 다음에는 해당 코드를 최대한 클린코드로 만들기 위해 여러번 고민해야 된다는 것을 깨달았다.
